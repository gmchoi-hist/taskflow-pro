# 01. Product

## 목표

팀 업무를 시각화한다. "지금 누가 뭐해?"라는 질문이 나오지 않는 것을 목표로 한다.

## 페르소나

- 10명 규모 스타트업의 팀 리더 (30~40대)
- 팀원 개개인의 작업 현황을 실시간으로 파악하고 싶어하지만, 무겁고 복잡한 툴을 배우거나 관리할 시간은 없음
- 빠르게 훑어보고 바로 판단할 수 있는 화면을 원함

## MVP 범위

### CRUD 4종 (화면에서 모두 동작)

- **추가**: 태스크 생성
- **목록**: 태스크 목록 조회
- **수정**: 태스크 내용/상태/담당자 등 수정
- **삭제**: 태스크 삭제

### 상태 분류

- 태스크는 상태값으로 분류되어 목록에서 구분 표시된다 (예: 할 일 / 진행 중 / 완료)

### 마감 시각 (due_at)

- 마감은 날짜만이 아니라 **시각까지** 지정한다
- 표시 형식 예: `2026-05-12 18:00`
- 생성/수정 화면에서 날짜와 시간을 함께 입력받는다

### 테마 토글

- 라이트/다크 테마를 토글 버튼으로 전환
- 선택한 테마는 `localStorage`에 저장되어 새로고침 후에도 유지

### 모바일 반응형

- 최소 360px 너비까지 레이아웃이 깨지지 않아야 한다

## UI 톤

Mac OS 스타일을 따른다.

- 둥근 모서리 (rounded corners)
- 부드러운 그림자 (soft shadow)
- 반투명 카드 (translucent card, backdrop blur)
- 시스템 폰트 사용 (San Francisco / 시스템 기본 폰트 스택)

## 확장 (MVP 이후, 우선순위 미정)

- JWT 로그인
- 팀 기능
- Kanban 보드
- 채팅
- CI/CD

## 범위 외 (Out of Scope)

- 외부 결제
- 네이티브 앱
- WebRTC
- 외부 캘린더 연동
- 파일 업로드
- WebSocket — MVP는 폴링(polling) 방식 사용, 실시간 갱신은 확장 단계에서 보류

## 성공 기준

- 새로고침해도 테마 설정이 유지된다
- 360px 너비에서 레이아웃이 깨지지 않는다
- API 응답 속도가 200ms 이내이다
- CRUD 4종(추가/목록/수정/삭제)이 화면에서 정상 동작한다
- 테마 토글이 정상 작동한다

## 참고
