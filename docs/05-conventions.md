# 05. Conventions

프로젝트 전반에 적용되는 코딩 컨벤션, 커밋 규칙, 네이밍 규칙을 정의한다. 이 문서는 가장 안정적으로 유지되며, 변경 시 팀 전체 합의가 필요하다.

## 언어 규칙

- 코드 식별자(변수, 함수, 클래스, 파일, 타입명)는 영어로 작성한다.
- 주석만 한국어로 작성한다. 커밋 메시지도 한국어 요약을 포함한다(아래 커밋 규칙 참고).
- 주석은 "왜"가 자명하지 않을 때만 작성한다. 코드가 이미 설명하는 내용(무엇을 하는지)은 주석으로 반복하지 않는다.

## 명명 규칙

| 대상 | 규칙 | 예시 |
|---|---|---|
| 백엔드 (Python) 변수/함수 | snake_case | `get_tasks`, `due_at` |
| 프론트엔드 (JS) 변수/함수 | camelCase | `fetchTasks`, `renderApp` |
| 컴포넌트/클래스 | PascalCase | `Task`, `TaskStatus`, `TaskCard` |
| 상수 | UPPER_SNAKE_CASE | `POLLING_INTERVAL_MS`, `DATABASE_URL` |

- API 요청/응답 JSON 필드(`due_at` 등)는 백엔드 규칙(snake_case)을 그대로 따르며, 프론트엔드에서 별도로 camelCase 변환을 하지 않는다. 이는 데이터 필드명이지 프론트엔드 코드의 변수/함수 식별자가 아니기 때문이다.

## 금지 사항

| 금지 | 이유 | 대안 |
|---|---|---|
| `print` 디버깅 | 로그 레벨 구분 불가, 운영 환경에서 노이즈 발생 | `logging` 모듈 사용 |
| bare `except` | 의도하지 않은 예외까지 삼켜 버그를 은폐함 | `except SpecificError:` 처럼 예외 타입을 명시 |
| 비밀번호/시크릿 하드코딩 | 코드 유출 시 보안 사고로 직결 | `.env` + `os.getenv()`로 관리 |
| TypeScript `any` 타입 | 타입 정보가 사라져 컴파일 타임 검증 무의미 | 명시적 타입 선언 |
| CSS `!important` | 우선순위 꼬임으로 이후 스타일 수정이 어려워짐 | 셀렉터 구조 개선으로 우선순위 문제 해결 |

## 테스트

- 백엔드는 `pytest`로 작성한다.
- 각 API/기능은 최소한 다음 케이스를 포함한다.
  - 정상 케이스 (성공 응답)
  - 실패 케이스: 검증 실패(400), 리소스 미존재(404)
- 테스트가 작성되고 통과해야 작업을 완료로 본다(CLAUDE.md 절대 규칙 3).

## 커밋 규칙

- 형식: `<type>: <한국어 요약>`
- type 종류: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`
- 예시
  - `feat: 태스크 생성 API 추가`
  - `fix: due_at 형식 검증 누락 수정`
  - `docs: Phase 1 설계 문서 7종 작성`
  - `refactor: 태스크 조회 쿼리 정리`
  - `test: 태스크 삭제 API 테스트 추가`
  - `chore: 의존성 버전 고정`
- 한 커밋은 하나의 논리적 변경 단위로 유지한다 (기능 추가와 리팩터링을 한 커밋에 섞지 않음).

## 참고

- 이 컨벤션에서 다루지 않는 애매한 스타일 문제가 발생하면 임의로 결정하지 않고 사용자에게 선택지를 제시해 확인한다(CLAUDE.md 모호한 요청 처리 원칙).
